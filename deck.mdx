# What is Next.js?

Key features:

- File-system based routing
- Hybrid static & server rendering
- Incremental Static Generation
- Smart bundling
- Route pre-fetching

# File-system based routing

<Steps>

`my-next-project/pages/index.tsx ==> https://mywebsite.com`

`my-next-project/pages/about.tsx ==> https://mywebsite.com/about`

`my-next-project/pages/posts/[id].tsx ==> https://mywebsite.com/posts/:id`

# Hybrid static & server rendering

- Build static pages when you can, render rest on server
- Controlled by exporting a function named `getStaticProps` or `getServerSideProps` from within a _page_
- Dynamic routes can also be statically generated, by exporting a function named `getStaticPaths`

# Other cool stuff

<Steps>

- Webpack, Babel and PostCSS configs are customizable without any hacks or "ejecting"
- One button deployment with Vercel
- You can run the server programmatically
  - Add your own express middlewares and whatnot
  - But you lose the one button deploy and automatic static optimization
- Serverless function support

</Steps>
